Idea behind IdentiChip

As the developer and administrator of www.getmeknown.co.za and www.getmehome.co.za it's become abundantly clear that the Microchip / Tag / Animal Identification world is in dire need of a easy to use single point of search.


And that's what IdentiChip tries to solve. By allowing providers to register and open their data through a custom endpoint and have their results included in the search.

Current Workflow for finding animal information

  • You find a lost animal and scan it for it's UID
  • Great, now where do you search for details on this microchip ?
  • Each provider has it's own database and search implementation. So do you visit each provider ?
  • Do you know all providers that cover you area ?
  • Google them ? Sure you'll only find all the providers that matched your query !
  • There is no single regonized search site / app, which is exactly the problem.

The workflow IdentiChip is aiming for

  • You find a lost animal and scan it for it's UID
  • You head to identichip.org or a app using the api and do the search.
  • IdentiChip searches all registered providers and gives the user details on the animal
  • The owner is contacted and the animal is returned home.
  • Simple, quick and easy to use. Why should we not have this in todays age?

So that's why I built IdentiChip

Provider Definition

A Provider represents a single service supplying and handling data for a database of animals, of any species.

To be listed on the site a provider must be approved and tested.

Approval Providers are approved by our administrators that do basic checks to see if the supplied information is valid and the registering provider is actually a provider.
Tested Providers are marked as tested when they have passed one validation test from our site. Which will also configure their API url.

So whats the Plan

  • We need a website / api where users / clients can search all these providers
  • Providers should be able to register and be approved in a simple and compelling way.
  • Instructions are provided to assist you in using this presentation template effectively
  • At all times strive to maintain Google's corporate look and feel

IdentiChip API

The most important part of IdentiChip, apart from single location for search.

The API allows third-party applications to:

  • Perform a Search
  • List Registered and Approved Providers
  • Include your Provider in the Searches
  • Pull Search Details for use to list searches of a UID for a provider
  • Pull search statistics

Here's a quick walk through

Building your Provider Endpoint

Each provider has to build a endpoint if they wish to be included in the search.

Our service authenticates with a hash generated from a secret key given to your provider.

A search is done by asynchronously calling all the endpoints of our providers with a 4 seconds timeout.

Example of a Successful Response

{
    "uid":"xxxxxxxxx",
    "name":"Bruno",
    "gender":"male",
    "distinctive_features":"Some Descriptive Features of Pet",
    "dob":"2012",

Example of a Not Found Response

{
  "search_result":"false",
  "message":"Could not find that Tag"
}

Example of a Failed Response

{
  status: "failure",
  message: "No Access Key Provided"
}
