Let's get started, what is App Engine ?

App Engine Allows you to run apps on Google's infrastructure

Every app gets free domains:


http://{yourappid}.appspot.com


https://{yourappid}.appspot.com


And register for Google Apps to use your custom domain. Both your Naked A record and CName's

And you don't have to worry that the free tier of Apps is not available anymore.

Register for Apps through your App Engine dashboard and you get a free version too


But instead of 5 free users it gives you 1 user. So you can still use custom domains for free


Nice little hack they left in for us

Your python code runs in a sandbox on a optimized python interpreter

This means some of your functions are limited

You can't open a socket or write to the file system


And any libraries / functions that attempt to use these functions will throw a exception

The dreaded billing problem

but let's not look at that way !

Google gives you a free qoute of 28 frontend hours and 1GB of storage on the Datastore.


Other service API's have their own quotas. Like Mail giving you 100 mails free everyday.

Grow big before you pay


All applications have a default quota configuration, the "free quotas", which should allow for roughly 5 million pageviews a month for an efficient application.

What does a Python App look like ?


This could be different for everyone as you can choose, use either webapp2 or Django. 


application: myapp
version: 1
runtime: python27
api_version: 1
threadsafe: true

Every App Engine Project has a app.yaml


This file defines your application key, the current version various of other options

Setting up Handlers



In this same file you also define handlers with 


handlers:
- url: /
  script: home.app


handlers:
- url: /stylesheets
  static_dir: stylesheets

And you can filter the these urls with /(.*\.(gif|png|jpg))

Great so how I do deploy my app ?

Included in your SDK are quite a few tools work with your application


With the biggest for our demo being:


dev_appserver.py


Local Dev environment with a few dozen arguments to configure local MySQL / Blobstore etc


appcfg.py


Used to update, download bulk data, upload bulk data, download your app code

Authenticates to your Google account with OAuth. Just add --oauth2

appcfg.py update {folder-name}/ --oauth2

So now that your "app" is running. Last thing to mention would be versions.

You have 2 options when splitting traffic



IP Based:


Assigns a hash to the IP of the user and serves the app based on it


Cookie Based:


Assigns a hash and puts it in a cookie for your app which means even if the IP of the user changes they will always get the same version while they have the cookie assigned

So taking another look at your URL:


{yourappid}.appspot.com


and to access a certain version:


{version}.{yourappid}.appspot.com
